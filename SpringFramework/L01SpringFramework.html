Spring Framework

1. 프레임워크(Framework)란?

  • 정의: 개발자가 효율적으로 프로그램을 만들 수 있도록 구조(틀)와 공통 기능을 제공하는 개발 도구.
  • 특징: 일정한 규칙과 구조를 강제하여 일관된 개발을 가능하게 함.
  • 장점: 생산성 향상, 유지보수 용이, 재사용성 증가, 중복 코드 감소.
  • 예시:
    • 웹: Spring, Django, Express.js
    • 데스크탑: JavaFX, Qt
    • 게임: Unity, Unreal

2. 스프링 프레임워크(Spring Framework)란?

  • 정의: 자바 기반의 엔터프라이즈 애플리케이션을 쉽고 효율적으로 개발하기 위한 오픈소스 프레임워크.
  • 핵심 철학:
    • POJO (Plain Old Java Object): 순수한 자바 객체 중심의 개발.
    • IoC (Inversion of Control): 객체의 생성과 생명주기를 스프링 컨테이너가 관리.
    • DI (Dependency Injection): 필요한 객체를 코드에서 직접 생성하지 않고 외부에서 주입받음.
    • AOP (Aspect Oriented Programming): 공통 기능(로깅, 트랜잭션 등)을 핵심 로직과 분리.
  • 주요 모듈: Core, Context, AOP, JDBC, ORM, Web, MVC 등.

3. 스프링으로 개발할 수 있는 아키텍처 종류

  • 1) Spring Core: IoC, DI 기반의 기본 컨테이너 기능 제공.
  • 2) Spring MVC: 웹 애플리케이션용 Model-View-Controller 구조.
  • 3) Spring WebFlux: 비동기(Non-blocking) 기반 Reactive 아키텍처.
  • 4) Spring Data: JPA, MongoDB 등 다양한 DB 연동.
  • 5) Spring Security: 인증·인가(로그인, 접근제어) 처리.
  • 6) Spring Batch / Integration: 대용량 데이터 처리, 시스템 간 연동.
  • 7) Spring Boot: 설정을 자동화하여 애플리케이션을 빠르게 실행.

4. 스프링의 객체 관리: Factory, DI, Singleton

  • 1) Factory: 객체를 생성하고 관리하는 ‘공장’ 역할의 클래스 또는 컨테이너.
    → Spring에서는 ApplicationContext가 Factory 역할을 수행.
  • 2) Dependency Injection (의존성 주입):
    • 필요한 객체를 직접 new로 생성하지 않고 외부에서 주입.
    • XML 설정, Annotation(@Autowired, @Inject), Java Config 등으로 주입 가능.
  • 3) Singleton 패턴:
    • 스프링 컨테이너는 기본적으로 각 Bean을 한 번만 생성하고 재사용함.
    • 동일한 Bean을 여러 곳에서 주입해도 하나의 인스턴스를 공유.

5. 스프링 웹과 MVC 구조

  • MVC란? Model, View, Controller로 구성된 디자인 패턴.
    • Model: 비즈니스 로직과 데이터 처리 (Service, DAO 등).
    • View: 사용자에게 출력되는 화면 (HTML, JSP, Thymeleaf 등).
    • Controller: 사용자의 요청을 받아 로직(Model)을 호출하고 결과를 View에 전달.
  • Spring MVC 동작 순서:
    1. 사용자가 URL 요청 → DispatcherServlet이 요청을 받음.
    2. HandlerMapping이 해당 요청을 처리할 Controller 찾음.
    3. Controller가 Service를 호출해 비즈니스 로직 수행.
    4. 결과를 Model에 담아 ViewResolver에게 전달.
    5. ViewResolver가 JSP 또는 템플릿(View) 렌더링 후 사용자에게 응답.

6. 스프링 부트(Spring Boot)와의 관계

  • Spring Boot: 스프링 프레임워크를 쉽게 사용하도록 만든 확장 프레임워크.
  • 특징:
    • 자동 설정(Auto Configuration)
    • 내장 서버(Tomcat, Jetty) 내장 → 바로 실행 가능
    • starter dependency를 통한 의존성 관리 단순화
  • 목표: 설정보다 개발 중심으로, 빠르게 프로토타입을 만들고 배포 가능하게 함.
