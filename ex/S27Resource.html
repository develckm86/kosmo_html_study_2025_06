경로와 iframe

1. 외부 스타일시트 (External CSS)

  • <link> 태그로 CSS 파일을 연결
  • 작성 위치: 보통 <head> 안
  • 여러 HTML 문서에서 재사용 가능, 유지보수 용이
  • 예시: 
    <head>
  <meta charset="UTF-8">
  <link rel="stylesheet" href="style.css">
</head>

2. 내부 스타일시트 (Internal CSS)

  • <style> 태그 안에 CSS 작성
  • 작성 위치: 보통 <head> 안, 외부 스타일 및
  • 해당 HTML 문서에만 적용
  • 예시: 
    <head>
  <style>
    body { background: lightblue; }
  </style>
</head>

3. 인라인 스타일 (Inline CSS)

  • 요소의 style 속성에 직접 작성
  • 작성 위치: 해당 태그 내부 속성
  • 특정 요소 하나에만 적용, 유지보수 불편
  • 예시: 
    <p style="color: red; font-size: 20px;">인라인 예시</p>

4. 적용 우선순위

  • 인라인 > 내부 > 외부
  • 같은 속성이 충돌하면 더 가까운(우선순위 높은) 선언이 적용

1. 절대경로 (Absolute Path)

  • 루트(최상위)에서 시작하는 전체 경로
  • 운영체제 경로나 프로토콜(http/https)까지 포함된 전체 URL
  • 예시: 
    
/home/user/project/index.html        (Linux)
/Users/경민/Documents/file.txt       (Mac)
C:\Users\경민\Desktop\test.html      (Windows)
http://localhost:63342/html_study/L26Resource.html
https://example.com/assets/css/style.css

2. 상대경로 (Relative Path)

  • 현재 문서가 있는 위치를 기준으로 리소스를 찾는 경로
  • 종류:
    • ./ : 현재 폴더 기준 예) ./style.css → 현재 폴더의 style.css
    • ../ : 상위 폴더 기준 예) ../img/cat.png → 상위 폴더의 img/cat.png
    • 파일명 : 같은 폴더 안의 파일 예) script.js → 현재 폴더의 script.js
    • / : 루트 상대경로 (서버 최상위 기준) 예) /src/img/cat.png → 서버 루트(/) 기준으로 src/img/cat.png

3. 비교

  • 절대경로: 언제나 같은 위치를 가리킴 (주소 고정)
  • 상대경로: 현재 문서 위치에 따라 달라짐 (유연하지만 이동에 취약)
  • 루트 상대경로(/): 서버 루트 기준으로 찾기 때문에 프로젝트 최상단 구조를 기준으로 일정함

CORP (Cross-Origin Resource Policy)

1. 개념

  • CORP는 다른 출처(origin)에서 불러오는 리소스를 브라우저가 어떻게 다룰지 결정하는 보안 정책
  • 기본 보안 정책인 SOP(Same-Origin Policy)를 보완하는 역할
  • 리소스 제공자가 “이 리소스는 어디서 불러 쓸 수 있는가?”를 명시

2. 설정 방법

  • 서버가 응답 헤더에 다음과 같이 추가
    • Cross-Origin-Resource-Policy: same-origin

3. 주요 옵션

  • same-origin : 같은 출처에서만 리소스 사용 가능
  • same-site : 동일 사이트 내 (예: example.com 과 sub.example.com) 허용
  • cross-origin : 어떤 출처에서든 사용 가능

4. 동작 예시

  • 이미지가 https://example.com/img.png에 있을 때
  • 헤더: Cross-Origin-Resource-Policy: same-origin
  • https://example.com/page.html → 정상 표시
  • https://other.com/page.html → 차단됨

5. 필요성

  • 민감한 리소스가 다른 사이트에 무단으로 포함되는 것 방지
  • 데이터 유출 차단
  • COEP, CORS 등 다른 정책과 함께 사용하여 보안을 강화

페이지 안의 작은 브라우저 창 iframe

1. 개념

  • <iframe>은 HTML 문서 안에 또 다른 HTML 문서나 외부 웹페이지를 삽입할 때 사용하는 태그
  • “페이지 안의 작은 브라우저 창” 역할을 함

2. 기본 문법

  • 형식: 
    <iframe src="주소" width="가로크기" height="세로크기"></iframe>
  • src: 불러올 문서(URL 또는 경로)
  • width, height: 프레임 크기 지정 (px 또는 %)

3. 주요 속성

  • src : 불러올 문서의 경로
  • title : 프레임 제목 (접근성 향상용)
  • frameborder : 테두리 표시 여부 (0: 없음, 1: 있음)
  • allowfullscreen : 전체 화면 허용
  • loading="lazy" : 화면에 보일 때만 로드 (성능 최적화)
  • referrerpolicy : 참조 정보 전달 정책
  • sandbox : 보안 옵션 (스크립트 실행, 폼 제출, 팝업 등 제한)

4. 경로 사용 예시

  • 상대경로 
    <iframe src="./L01Hello.html"></iframe>
  • 루트 상대경로 
    <iframe src="/L01Hello.html"></iframe>
  • 절대경로(URL) 
    <iframe src="http://localhost:63342/html_study/L01Hello.html"></iframe>
  • 외부 사이트 
    <iframe src="https://www.naver.com"></iframe>

5. 활용 예시

  • 내부 문서 포함하기
    • <iframe src="sub/page.html" width="400" height="300"></iframe>
  • 외부 사이트 포함하기
    • <iframe src="https://example.com" width="600" height="400"></iframe>
  • YouTube 영상 임베드
    • <iframe width="560" height="315"
  src="https://www.youtube.com/embed/동영상ID"
  title="YouTube video player"
  frameborder="0"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
  allowfullscreen></iframe>
