웹 개발 기초 개념

1. 인터넷(Internet)

  • 전 세계의 컴퓨터와 네트워크가 연결된 거대한 정보 통신망
  • TCP/IP 프로토콜을 기반으로 데이터가 패킷 단위로 전송됨
  • 인터넷 위에서 다양한 서비스(웹, 이메일, FTP 등)가 동작함

2. 웹(Web)

  • 인터넷에서 정보를 주고받는 대표적인 서비스
  • HTTP(또는 HTTPS) 프로토콜을 이용해 문서(HTML)를 주고받음
  • 브라우저는 HTML, CSS, JavaScript를 해석하여 사용자에게 시각적으로 보여줌

3. 웹서버(Web Server)

  • 클라이언트(브라우저)의 요청을 받아 정적인 콘텐츠를 제공하는 서버
  • 예: HTML, CSS, JS, 이미지 파일 등
  • 대표 소프트웨어: Apache, Nginx, IIS
  • 주요 역할:
    • HTTP 요청 처리
    • 정적 파일 제공
    • 요청을 웹 애플리케이션 서버(WAS)로 전달

4. 웹 애플리케이션 서버(WAS, Web Application Server)

  • 동적인 콘텐츠(프로그램 실행 결과)를 생성하는 서버
  • 웹서버와 달리 비즈니스 로직을 실행하고 DB와 연동 가능
  • 대표 예시: Tomcat, JBoss, WebLogic, Jetty
  • 주요 역할:
    • JSP, Servlet, Spring 등의 실행 환경 제공
    • DB 연동 및 트랜잭션 처리
    • 세션 관리 및 보안 처리

5. 웹 애플리케이션(Web Application)

  • 웹서버와 WAS 위에서 동작하는 실제 서비스 프로그램
  • 사용자 요청에 따라 DB에서 데이터를 조회하고, HTML로 응답을 생성
  • 예시: 쇼핑몰, 블로그, 일정 관리 시스템, 온라인 은행 등
  • 구성 요소:
    • 프론트엔드: HTML, CSS, JavaScript로 구성
    • 백엔드: 서버 언어(Java, Python, Node.js 등)와 DB 연동

6. 웹 구조의 흐름


[브라우저] → [웹서버] → [WAS] → [DB]
   요청         전달        로직처리    데이터조회
   ← 응답       ← 결과       ← 결과HTML
